[Jacob A. Sonnenberg]

2019/10/20

*Well not really the beginning

This is a post overdue, to a blog I should have started a while ago, if not to achieve anything other than a “I could so I did” thing. After multiple failed attempts using blogger, wordpress, et al. I stopped because I didn’t really like the workflow. 1

So I’m not one to bear the burden of a over-heavy work-flow, if it should take much work to spew my words into the void I likely won’t. So the work-flow is vitally important!

With a combination of Emacs’ org-capture, plus the ox-hugo and easy-hugo interfaces for the lovely static site generator hugo and a stray deploy.sh script I’ve got the publishing work-flow down to

C-c c b
C-c C-e H H
C-c b
G

Where C-c c b is my capture template for a blog post, captured to ~/org/blog.org. This holds all my posts in a single org file defined by defined by my org-capture template2. I write everything here, or go back to the file to make edits. But :prepend t and ~:jump-to-captured t make it easy3 to find.

("b" "Blog post" entry
 (file "~/Documents/org/blog.org")
 "* %^{Title}
:PROPERTIES:
:EXPORT_FILE_NAME: %^{Export File Name}
:EXPORT_DATE: %U
:END:
%?" :prepend t :jump-to-captured t :empty-lines-after 1)

C-c C-e H H exports the sub-tree as markdown to the hugo directory thanks to ox-hugo.

(use-package ox-hugo :after ox :ensure t)

C-c b calls easy-hugo for a comprehensive Emacs interface. From here I can G to publish by running deploy.sh which uploads the newly generated site to github pages. Which hugo has a good tutorial for.

And so the spell is cast! With a invocation of ten (chorded) keystrokes.
